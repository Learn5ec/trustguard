🔍 Security Findings 1 HIGH · 1 MEDIUM · 1 INFO
1. CSRF vulnerability in older versions
HIGH
DEPENDENCY_CVE ✓ Confirmed
FastAPI versions before 0.65.2 were vulnerable to CSRF attacks when using cookies for authentication in JSON payload operations. The vulnerability allowed browsers to send JSON payloads with text/plain content-type, bypassing CORS protections. This could lead to unauthorized actions being performed on behalf of authenticated users.
Vulnerability GHSA-8h2j-cgx8-6xv7 in provided vulnerability data, affecting versions <0.65.2. Fixed in version 0.65.2 via PR #2118.
Ensure you are using FastAPI version 0.65.2 or later. If stuck on an older version, implement a middleware to validate content-type headers.
2. ReDoS vulnerability in form data parsing
MEDIUM
DEPENDENCY_CVE ✓ Confirmed
FastAPI versions before 0.109.1 were vulnerable to Regular Expression Denial of Service (ReDoS) when parsing maliciously crafted Content-Type headers in form data. This could allow attackers to stall the event loop and cause service disruption.
Vulnerability PYSEC-2024-38 in provided vulnerability data, affecting versions <0.109.1. Fixed in commit 9d34ad0ee8a0dfbbcce06f76c2d5d851085024fc.
Upgrade to FastAPI version 0.109.1 or later to mitigate this vulnerability.
